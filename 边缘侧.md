# Orin 边缘侧部署指南

## 环境要求
| 项目 | 要求 |
|------|------|
| 平台 | NVIDIA Jetson Orin Nano / NX / AGX |
| 系统 | JetPack 5.x（CUDA 11.4 + TRT 8.x）或 JetPack 6.x（CUDA 12.x + TRT 10.x） |
| Python | >= 3.10 |
| 磁盘 | >= 32GB |
| 显存 | >= 8GB |
## 系统依赖
JetPack 通过 SDK Manager 或 `apt` 安装，包含 CUDA / TensorRT / CuDNN。验证：
```bash
nvcc --version && dpkg -l | grep tensorrt
```
ffmpeg（H.264 推流必需）：
```bash
sudo apt install -y ffmpeg
```
## 项目依赖
faiss-cpu 在 aarch64 上无预编译轮子，`pip install` 会源码编译，需先装编译依赖：
```bash
sudo apt install -y swig libblas-dev liblapack-dev
# Orin Nano 8GB 内存可能不够，加 swap：
sudo fallocate -l 4G /swapfile && sudo chmod 600 /swapfile
sudo mkswap /swapfile && sudo swapon /swapfile
```
然后安装项目运行所需依赖：
```bash
cd /path/to/YZ-boat-orin
python3 -m pip install --upgrade pip setuptools wheel
python3 -m pip install -e .
```
TensorRT / PyCUDA 随 JetPack 安装，不在 pyproject.toml 中。若缺 pycuda：
```bash
pip install pycuda
# 编译失败时：export CUDA_PATH=/usr/local/cuda
```
> `pip install -e .` 会安装当前 `pyproject.toml` 声明的运行依赖；`ultralytics` 不在项目依赖中，仅在导出 `.engine` 时单独安装。
> 非 Orin 环境无 TensorRT 时，程序调用检测器才报错，其他功能不受影响。
## YOLO Engine 导出
```bash
pip install ultralytics  # 仅导出时需要
yolo export model=yolov8n.pt format=engine device=0 half=True imgsz=640
```
> `.engine` 与 GPU 架构绑定，**务必在目标 Orin 上导出**，x86 导出的无法使用。
## VLM 服务部署
> Orin 显存有限，VLM 通常部署在远程 GPU 服务器上，Orin 通过网络调用。
```bash
# GPU 服务器上
pip install "vllm==0.11.1" --torch-backend=cu128
CUDA_VISIBLE_DEVICES=0 vllm serve /path/to/Qwen3-VL-4B-AWQ \
  --api-key abc123 --served-model-name Qwen/Qwen3-VL-4B-AWQ \
  --max-model-len 10240 --port 7890 --gpu-memory-utilization 0.15 --max-num-seqs 10

# Embedding 服务（可选）
CUDA_VISIBLE_DEVICES=1 python -m vllm.entrypoints.openai.api_server \
  --model ./models/Qwen3-Embedding-0.6B --api-key abc123 \
  --served-model-name Qwen3-Embedding-0.6B --convert embed \
  --gpu-memory-utilization 0.08 --max-model-len 2048 --port 7891
```
VLM 部署远程时，修改 `config.yaml` 中的 `base_url`：
```yaml
llm:
  base_url: "http://<VLM-IP>:7890/v1"
  api_key: "abc123"
embed:
  base_url: "http://<VLM-IP>:7891/v1"
```
## 启动
**Web 界面**（当前仓库 `config.yaml` 默认端口是 `8000`；如果要用 `9000`，请同步修改配置或显式传参。`--ws-ping-interval 0` 必须设置，否则推流断连）：
```bash
uvicorn web.app:app --ws-ping-interval 0 --host 0.0.0.0 --port 9000
```
**CLI 命令行**：
```bash
python -m pipeline.cli <video> --demo --output result.mp4       # 视频
python -m pipeline.cli 0 --demo --display                        # 摄像头
python -m pipeline.cli rtsp://... --demo --display              # RTSP
python -m pipeline.cli <video> --demo -c --max-concurrent 4     # 并发
python -m pipeline.cli <video> --demo --max-frames 50 -v        # 快速测试
```
## 关键配置 (config.yaml)
```yaml
pipeline:
  yolo_model: "yolov8n.engine"    # TensorRT 引擎
  device: "0"                       # GPU 设备号
  conf_threshold: 0.25
  detect_every_n_frames: 2
  detect_classes: [8]               # COCO 8=boat
  concurrent_mode: true
  max_concurrent: 4
camera_demo:
  stream_mode: "h264"
web:
  host: "0.0.0.0"
  port: 9000
```
> 若直接使用仓库当前默认配置，`web.port` 仍是 `8000`；上面示例表示部署时建议改为 `9000`。
## 环境变量
| 变量 | 说明 | 默认 |
|------|------|------|
| `TRT_DETECTOR_DEBUG` | `1` 开启 TRT 调试 | 关 |
| `TRT_SHIP_CLASS_IDS` | `all` 或逗号分隔 ID | `8` |
| `TRT_YOLO_HAS_OBJECTNESS` | `1` 启用旧模型解码 | 关 |
| `CUDA_VISIBLE_DEVICES` | 限制可见 GPU | 全部 |
## 端口
| 端口 | 服务 |
|------|------|
| 9000 | Web 管理界面 |
| 7890 | VLM 推理（远程） |
| 7891 | Embedding（远程，可选） |
## 常见问题
| 问题 | 解决 |
|------|------|
| `No module 'tensorrt'` | `sudo apt install python3-libnvinfer` |
| `No module 'pycuda'` | `pip install pycuda`，编译失败设 `CUDA_PATH=/usr/local/cuda` |
| `faiss-cpu` 编译失败 | `sudo apt install -y swig libblas-dev liblapack-dev`，8GB 内存加 swap |
| `.engine` 加载失败 | TRT 版本不兼容，在**当前 Orin** 上重新 `yolo export` |
| WebSocket 推流断连 | 确保有 `--ws-ping-interval 0`，检查网络/负载 |
| VLM 超时 | 确认 `curl http://<VLM-IP>:7890/v1/models` 可达 |
| ffmpeg 错误 | `which ffmpeg && which ffprobe` 确认已安装 |
