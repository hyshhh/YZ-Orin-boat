## WebSocket推流协议

### 数据格式

```
┌─────────────────────────────────────────────────────┐
│                    WebSocket 帧                      │
├─────────────────────────────────────────────────────┤
│  字节数   │  偏移    │  说明                        │
├─────────────────────────────────────────────────────┤
│  1        │  0       │  消息类型                    │
│           │          │   0x01 = 初始化段 (moov)     │
│           │          │   0x02 = 媒体段 (moof+mdat)  │
├─────────────────────────────────────────────────────┤
│  4        │  1       │  数据长度 (uint32 big-endian)│
├─────────────────────────────────────────────────────┤
│  N        │  5       │  fMP4 数据                   │
└─────────────────────────────────────────────────────┘

示例:
初始化段: [0x01][00 00 0A E8][moov box data...]
媒体段:   [0x02][00 01 2A F0][moof+mdat data...]
```

## 错误处理

### 常见错误

1. **TensorRT 导入失败**
   - 原因: 未安装 TensorRT 或 PyCUDA
   - 解决: `pip install tensorrt pycuda`

2. **ffmpeg 不可用**
   - 原因: 未安装 ffmpeg
   - 解决: `apt install ffmpeg` 或下载二进制

3. **摄像头打开失败**
   - 原因: 设备被占用或不存在
   - 解决: 检查设备号, 关闭其他程序

4. **WebSocket 连接失败**
   - 原因: 任务不存在或已结束
   - 解决: 检查任务状态, 重新启动

5. **H264 播放黑屏**
   - 原因: ffmpeg 编码失败
   - 解决: 检查 ffmpeg 日志, 调整参数

```javascript
ws.onmessage = (evt) => {
  if (evt.data instanceof ArrayBuffer) {
    const view = new DataView(evt.data);
    const msgType = view.getUint8(0);      // 0x01 或 0x02
    const length = view.getUint32(1);       // 数据长度
    const payload = evt.data.slice(5);      // fMP4 数据

    if (msgType === 0x01) {
      // 初始化段 - 创建 SourceBuffer
      sourceBuffer.appendBuffer(payload);
    } else if (msgType === 0x02) {
      // 媒体段 - 追加到 SourceBuffer
      sourceBuffer.appendBuffer(payload);
    }
  }
};
```
